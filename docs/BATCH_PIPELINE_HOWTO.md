# Batch Pipeline — Complete How-To Guide

The Batch Pipeline generates a pool of AI character candidates using your LLM, lets you review and approve them, then renders their images through ComfyUI. The pipeline has five distinct stages — this guide walks through all of them.

```
Generate Batch → Review Candidates → Save Character → Compile Prompts → Queue ComfyUI → Gallery
```

---

## Prerequisites

All of these must be in place before you start.

| What | Where | Notes |
|---|---|---|
| Dev server running | Terminal | `npm run dev` → http://localhost:5173 |
| LM Studio running | LM Studio app | Must be serving on a local port |
| `LMSTUDIO_BASE_URL` | `.env.local` | e.g. `http://localhost:1234` |
| `LMSTUDIO_MODEL` | `.env.local` | Exact model name shown in LM Studio |
| `ENABLE_CHARACTER_BATCH_API=true` | `.env.local` | Required for batch generation |
| `ENABLE_COMFY_API=true` | `.env.local` | Required for image generation |
| `COMFYUI_BASE_URL` | `.env.local` | e.g. `http://localhost:8188` |
| ComfyUI running with a workflow loaded | ComfyUI app | At least one valid workflow must exist |

> After any `.env.local` change, restart the dev server: `Ctrl+C` then `npm run dev`.

## Stage 1 — Generate a Batch

Open the **Casting Room** tab. You will see five numbered sections. The first is **Batch Pipeline**.

1. Click **+ Generate Batch** to expand the generation form.
2. Fill in the parameters:
   - **Age range** — min and max (e.g. 25 to 45). Leave blank for no constraint.
   - **Count** — number of candidates to generate (1–50). Start with 4–8.
   - **Gender** — filter by gender presentation, or leave blank for mixed.
   - **Tone** — optional creative direction, e.g. `gritty noir thriller` or `warm ensemble drama`. This is passed directly to the LLM.
3. Click **Generate**. LM Studio will produce the requested number of character profiles. This takes 15–90 seconds depending on count and model.

When generation completes, the batch is automatically selected in the dropdown and the candidates appear below.

> If generation fails, check Developer Tools (bottom of the page) for the Comfy status line and verify LM Studio is running. The error message in the red banner also shows the raw API error.

**Selecting an existing batch:** The dropdown at the top of the section shows all batches, labeled by date, count, and tone. Select any past batch to review its candidates.

---

## Stage 2 — Review Candidates

Each candidate card shows:

- **Name and age** (as generated by the LLM)
- **Review status**: `pending` / `approved` / `saved` / `rejected`
- **Similarity check**: `unique` (safe to add), `needs change` (close to an existing character), `too similar` (flagged as near-duplicate)
- **Cinematic archetype**

The buttons on each card change based on the candidate's current status:

| Status | Buttons shown | What they do |
|---|---|---|
| `pending` | Cast this character · Dismiss | Cast → marks approved. Dismiss → hides card (soft delete). |
| `approved` | Save → Active Character · Dismiss | Save → writes the character to the database. Dismiss → un-approves. |
| `saved` | Saved ✓ (no buttons) | Already in the database — move on to Stage 3. |
| `rejected` | Reconsider | Returns the candidate to `pending`. |

> **Dismissed candidates** are hidden by default. Click **Show dismissed (N)** at the top of the list to reveal them, then **Reconsider** to bring one back to `pending`.

Work through the candidates: Cast the ones you want to use, Dismiss the rest. You can Cast multiple candidates from the same batch.

---

## Stage 3 — Save as Active Character

Once a candidate is `approved`, click **Save → Active Character**.

- The character profile is written to the database permanently.
- The page scrolls down to the **Active Character** section (Section 3), where the new character is pre-selected.
- You can save multiple candidates from the same batch — each becomes its own character entry.

> At this point the character exists as a profile (name, age, face/hair/body attributes) but has **no images yet**. Stages 4 and 5 handle image generation.

---

## Stage 4 — Compile Prompt Packs

The Active Character section is directly below the Batch Pipeline section.

1. The character you just saved should already be selected in the dropdown. If not, select it.
2. Click **Compile Prompt Packs** (or **Recompile Packs** if packs already exist).
   - The app reads the character's attributes and the selected ComfyUI workflow, then compiles a prompt pack — one prompt per view angle.
   - The result: `Compiled N prompt pack(s).`
3. A prompt pack dropdown appears. Each pack corresponds to one render view.

> **What is a prompt pack?** It is the fully assembled ComfyUI prompt for one view of this character — positive prompt, negative prompt, sampler settings — all derived from the character profile and your workflow. No manual prompt writing needed.

> **Workflow selection**: The workflow dropdown at the top of the Portfolio section (Section 4) controls which workflow is used for compilation. If the list is empty, ComfyUI is not connected or no workflows are loaded. Check Developer Tools → Comfy status.

---

## Stage 5 — Queue the Portfolio (ComfyUI)

This is the step that actually sends work to ComfyUI.

Under **Portfolio** (Section 4):

1. Confirm the correct workflow is selected.
2. Tick the view angles you want to render:
   - `front portrait` — straight-on face, default on
   - `three quarter portrait` — default on
   - `profile portrait` — side view, default on
   - `full body` — default on
   - `audition still` — default on
   - `cinematic scene` — off by default (slower, more stylised)
3. Click **Queue Portfolio**.

The app sends one ComfyUI job per selected view. The success banner shows `Portfolio queued: N views.`

A job list appears under the button showing each view and its current status:

| Badge | Meaning |
|---|---|
| `pending` | Job is in the ComfyUI queue, not yet started |
| `running…` | ComfyUI is actively rendering |
| `completed` | Render finished, image ingested into Gallery |
| `failed` | Job failed — check ComfyUI logs |

The panel polls ComfyUI automatically every few seconds (the `⟳ checking Comfy…` indicator shows polling is active). Polling stops when all jobs reach a terminal state (completed or failed).

> **Switch tabs freely** — polling state is saved in `sessionStorage`. If you leave and return, the panel resumes polling for any pending jobs.

---

## Stage 6 — Review the Gallery

Images appear in **Gallery** (Section 5) as ComfyUI jobs complete.

The header shows **Showing images for: [Character Name]** to confirm which character's images you are viewing. Switching the Active Character dropdown updates the gallery immediately.

Each image card shows:
- View type and seed number
- Approval state: `✓` (kept) or `pending`

Actions:
- **Keep** — marks the image as approved (used when assembling final prompt sets)
- **Discard** — rejects it; the image data is kept in the database but flagged as rejected

Click **Refresh** to reload the gallery manually if the auto-ingest seems stalled.

## Saving Characters for Future Use

Characters saved via **Save → Active Character** are stored permanently in the SQLite database. They persist across dev server restarts and sessions.

To access a saved character in a future session:
1. Open the **Casting Room** tab.
2. Under **Active Character**, open the dropdown — all previously saved characters appear here.
3. Select one and click **Recompile Packs** to regenerate prompt packs for that character (needed if the workflow has changed or packs were lost).

To view a character's full profile and their reference images, use the **Actor Bank** tab. Every character saved through the Batch Pipeline appears there.

**Archive vs Delete:** Clicking **Archive** on a character hides it from the Active Character dropdown without deleting it. The character and all its images remain in the database. To restore an archived character, expand **▸ Archived characters** under the dropdown and click **Restore**.

---

## Generating More Images for an Existing Character

To add more view angles or get alternative renders for a character already in the database:

1. Select the character in **Active Character**.
2. Under **Portfolio**, tick the additional views you want (uncheck views you already have to avoid duplicates).
3. Click **Queue Portfolio** again. New jobs queue alongside any previous ones.

All results appear in Gallery as they complete.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Generate Batch fails with permissions error | Feature flag missing | Add `ENABLE_CHARACTER_BATCH_API=true` to `.env.local`, restart |
| Batch dropdown shows no batches | No batches generated yet | Click **+ Generate Batch** |
| All candidates show `too similar` | Existing characters in the DB are very close | Use `unique` candidates; similar ones can still be cast but will diverge once rendered |
| Compile Prompt Packs button disabled | No character selected | Select a character in the Active Character dropdown |
| Workflow dropdown is empty | ComfyUI not connected | Check Developer Tools → Comfy status line |
| Queue Portfolio button stays disabled | No character, no workflow, or no views ticked | Verify all three are set |
| Jobs stay at `pending` forever | ComfyUI not running or queue full | Check ComfyUI is running; check its queue |
| Gallery shows no images after jobs complete | Auto-ingest failed | Developer Tools → **Ingest Outputs** |
| Character missing from Active Character dropdown | Was archived | Expand **▸ Archived characters**, click **Restore** |
| Character not in Actor Bank | Not yet saved via Save → Active Character | Complete Stage 3 — Cast + Save |
